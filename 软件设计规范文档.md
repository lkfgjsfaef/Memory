## 1. 项目概述

### 1.1 项目简介

Memory 是一款面向情侣用户的记忆分享应用，旨在帮助情侣记录和分享恋爱中的美好瞬间。应用支持多端访问，包括 Web 网页端和 Android 移动端，共享同一套后端 API 服务。

### 1.2 核心功能

| 功能模块 | 功能描述 |
|---------|---------|
| **用户认证** | 用户登录/登出、JWT令牌认证、头像管理 |
| **首页/情侣信息** | 恋爱天数计算、情侣信息管理、重要日子倒计时 |
| **日常记录** | 图文日记、心情标记、地点记录、按月筛选 |
| **恋爱日历** | 月历视图、每日心情标记、日期标记备注 |
| **心愿清单** | 心愿创建/管理、分类筛选、状态跟踪、完成标记 |
| **回忆库** | 足迹地图、记忆长河时间线、时光相册 |
| **音乐播放** | 网易云音乐搜索、歌单管理、在线播放 |

### 1.3 用户角色

系统采用简化的双用户模型，预设两名情侣用户：

| 用户ID | 用户名 | 昵称 | 角色 |
|--------|--------|------|------|
| 1 | user1 | 用户一 | 他 |
| 2 | user2 | 用户二 | 她 |

> **设计说明**: 系统当前为单情侣对设计，数据在情侣间共享（通过 `user_id` 区分作者）。

## 4. 后端设计规范

### 4.1 包结构规范

```
com.niit.memory/
├── MemoryApplication.java              # 启动类
├── config/                             # 配置层（CORS、JWT、Web、密码编码器、七牛云、Result、UserContext）
├── controller/                         # 控制器层（按功能模块划分，一个Controller可管理多个子资源）
├── entity/                             # 实体层（纯POJO，与数据库表一一对应）
├── repository/                         # 数据访问层（JdbcTemplate + 参数化SQL）
└── service/                            # 业务逻辑层（不含UserRepository对应的Service）
```

**命名规范**：
- Controller: `{Feature}Controller`（如 `DailyController`, `MemoryController`）
- Service: `{Feature}Service`（如 `DailyRecordService`, `MemoryMomentService`）
- Repository: `{Entity}Repository`（如 `DailyRecordRepository`, `CalendarMoodRepository`）
- Entity: 与表名对应的驼峰类名（如 `DailyRecord`, `VisitedLocation`）

### 4.2 Controller 层规范

#### 4.2.1 类定义模板

```java
@RestController
@RequestMapping("/api/{resource-name}")     // 路径统一 /api/ 前缀，kebab-case
public class XxxController {

    private final XxxService service;       // 依赖声明为 final

    // 构造器注入（不用 @Autowired）
    public XxxController(XxxService service) {
        this.service = service;
    }
}
```

**规范要点**：
- 统一使用 `@RestController`（非 `@Controller`）
- 依赖声明为 `final`，通过构造器注入，**禁止 `@Autowired` 字段注入**
- URL 前缀统一 `/api/`，资源名用 kebab-case
- 一个 Controller 可管理多个相关子资源（如 `MemoryController` 管理 albums/moments/locations）

#### 4.2.2 方法签名规范

```java

    try {
        String status = body.get("status");

    } catch (Exception e) {
        return Result.error(e.getMessage());
    }
}

// ===== 删除资源 =====
@DeleteMapping("/{id}")

        service.delete(id);
        return Result.success();
    } catch (Exception e) {
        return Result.error(e.getMessage());
    }
}
```

**规范要点**：
- **所有方法返回 `Result`**，统一包裹在 try-catch 中
- `@PathVariable` 类型统一 `Long`
- `@RequestParam(required = false)` 用于可选过滤参数
- `@RequestBody` 接收实体对象或 `Map<String, String>`（部分更新场景）
- **更新操作**：路径 `id` 覆盖请求体的 `id`（`entity.setId(id)`）
- **创建成功**返回 `Result.success()`，**更新成功**返回 `Result.success(更新后对象)`
- **部分字段更新**使用 `@PatchMapping`，参数用 `Map<String, String>` 接收

```java
// ===== 列表查询（带过滤 + 业务计算）=====
public List<ImportantDate> findAllWithDaysLeft() {
    List<ImportantDate> dates = repository.findAll();
    LocalDate today = LocalDate.now();
    for (ImportantDate date : dates) {
        if (date.getRecurring() != null && date.getRecurring() == 1) {
            // 重复事件：计算下一次的倒计时
            LocalDate next = today.withMonth(date.getRecurringMonth())
                                  .withDayOfMonth(date.getRecurringDay());
            if (next.isBefore(today) || next.isEqual(today)) {
                next = next.plusYears(1);
            }
            date.setDaysLeft((int) ChronoUnit.DAYS.between(today, next));
        } else if (date.getEventDate() != null) {
            // 非重复事件：直接计算
            date.setDaysLeft((int) ChronoUnit.DAYS.between(today, date.getEventDate()));
        }
    }
    return dates;
}

// ===== 时间线分组（记忆瞬间按年月分组）=====
public List<Map<String, Object>> getTimeline() {
    List<MemoryMoment> all = repository.findAll();
    Map<String, List<MemoryMoment>> grouped = all.stream()
        .collect(Collectors.groupingBy(
            m -> m.getYear() + "-" + String.format("%02d", m.getMonth()),
            LinkedHashMap::new, Collectors.toList()));
    List<Map<String, Object>> timeline = new ArrayList<>();
    for (Map.Entry<String, List<MemoryMoment>> entry : grouped.entrySet()) {
        Map<String, Object> group = new HashMap<>();
        String[] parts = entry.getKey().split("-");
        group.put("year", Integer.parseInt(parts[0]));
        group.put("month", Integer.parseInt(parts[1]));
        group.put("moments", entry.getValue());
        timeline.add(group);
    }
    return timeline;
}

// ===== 全局共享数据（音乐歌单：全量替换）=====
public void savePlaylist(List<MusicSong> songs) {
    repository.deleteAll();
    for (int i = 0; i < songs.size(); i++) {
        MusicSong song = songs.get(i);
        song.setSortOrder(i);
        repository.save(song);
    }
}
```

- 统计方法返回 `Map<String, Object>`，key为驼峰命名
- 时间线分组使用 Java Stream `Collectors.groupingBy`
- 全局共享数据（如歌单）采用全量替换策略
- 业务计算（如恋爱天数、倒计时）在 Service 层完成，不出SQL

#### 4.4.2 查询方法

```java
// ===== 简单查询（无参数）=====
public List<DailyRecord> findAll() {
    String sql = "SELECT * FROM daily_record ORDER BY record_date DESC";
    return jdbc.query(sql, new BeanPropertyRowMapper<>(DailyRecord.class));
}

// ===== 带过滤的动态查询 =====
public List<DailyRecord> findByYearAndMonth(Integer year, Integer month) {
    StringBuilder sql = new StringBuilder("SELECT * FROM daily_record WHERE 1=1");
    List<Object> params = new ArrayList<>();
    if (year != null) {
        sql.append(" AND YEAR(record_date) = ?");
        params.add(year);
    }
    if (month != null) {
        sql.append(" AND MONTH(record_date) = ?");
        params.add(month);
    }
    sql.append(" ORDER BY record_date DESC");
    return jdbc.query(sql.toString(), new BeanPropertyRowMapper<>(DailyRecord.class), params.toArray());
}

// ===== 单条查询 =====
public DailyRecord findById(Long id) {
    String sql = "SELECT * FROM daily_record WHERE id = ?";
    List<DailyRecord> list = jdbc.query(sql, new BeanPropertyRowMapper<>(DailyRecord.class), id);
    return list.isEmpty() ? null : list.get(0);
}

// ===== 统计查询 =====
public int countAll() {
    String sql = "SELECT COUNT(*) FROM daily_record";
    Integer count = jdbc.queryForObject(sql, Integer.class);
    return count != null ? count : 0;
}
```

**规范要点**：
- 使用 `BeanPropertyRowMapper<>(XxxClass.class)` 自动映射（**要求数据库列名用下划线，Java字段用驼峰**）
- 动态条件拼接：`StringBuilder` + `WHERE 1=1` + `List<Object> params`
- 参数化查询：使用 `?` 占位符，`params.toArray()` 传参
- 单条查询：先 `jdbc.query` 获取 List，再判空取第一个

#### 4.4.3 增删改方法

```java
// ===== 插入 =====
public void save(DailyRecord record) {
    String sql = "INSERT INTO daily_record (title, content, author, user_id, location, mood, mood_icon, record_date, image_urls) " +
                 "VALUES (?, ?, ?, ?, ?, ?, ?, ?, ?)";
    jdbc.update(sql, record.getTitle(), record.getContent(), record.getAuthor(),
                record.getUserId(), record.getLocation(), record.getMood(),
                record.getMoodIcon(), record.getRecordDate(), record.getImageUrls());
}

// ===== 插入并返回自增ID =====
public long save(VisitedLocation location) {
    String sql = "INSERT INTO visited_location (name, province, visit_date, title, image_url, lat, lng) " +
                 "VALUES (?, ?, ?, ?, ?, ?, ?)";
    GeneratedKeyHolder keyHolder = new GeneratedKeyHolder();
    jdbc.update(connection -> {
        PreparedStatement ps = connection.prepareStatement(sql, Statement.RETURN_GENERATED_KEYS);
        ps.setString(1, location.getName());
        ps.setString(2, location.getProvince());
        ps.setDate(3, location.getVisitDate() != null ? Date.valueOf(location.getVisitDate()) : null);
        ps.setString(4, location.getTitle());
        ps.setString(5, location.getImageUrl());
        ps.setBigDecimal(6, location.getLat());
        ps.setBigDecimal(7, location.getLng());
        return ps;
    }, keyHolder);
    return keyHolder.getKey().longValue();
}

// ===== Upsert（插入或更新）=====
public void upsert(CalendarMood mood) {
    String sql = "INSERT INTO calendar_mood (mood_date, mood, mood_icon, user_id) VALUES (?, ?, ?, ?) " +
                 "ON DUPLICATE KEY UPDATE mood = VALUES(mood), mood_icon = VALUES(mood_icon)";
    jdbc.update(sql, mood.getMoodDate(), mood.getMood(), mood.getMoodIcon(), mood.getUserId());
}

// ===== 更新 =====
public void update(DailyRecord record) {
    String sql = "UPDATE daily_record SET title=?, content=?, author=?, location=?, mood=?, mood_icon=?, record_date=?, image_urls=? WHERE id=?";
    jdbc.update(sql, record.getTitle(), record.getContent(), record.getAuthor(),
                record.getLocation(), record.getMood(), record.getMoodIcon(),
                record.getRecordDate(), record.getImageUrls(), record.getId());
}

// ===== 删除 =====
public void deleteById(Long id) {
    String sql = "DELETE FROM daily_record WHERE id = ?";
    jdbc.update(sql, id);
}

// ===== 全量删除 =====
public void deleteAll() {
    jdbc.update("DELETE FROM music_playlist");
}
```

**规范要点**：
- 插入：使用 `jdbc.update(sql, params...)`，字段逐一列出（不用 `INSERT *`）
- 需要返回自增ID：使用 `GeneratedKeyHolder` + `PreparedStatement`
- Upsert：`INSERT ... ON DUPLICATE KEY UPDATE`，需配合唯一约束
- 更新：WHERE 条件放在最后，id 作为最后一个参数
- 日期类型：Java `LocalDate` → JDBC `Date.valueOf()`

### 4.5 Entity 层规范

```java
public class DailyRecord {

    private String location;
    private String mood;
    private String moodIcon;            // mood_icon → moodIcon

    private String imageUrls;           // 多图URL用逗号分隔存储为单个String
```

#### 4.6.3 JwtUtil — JWT工具类

```java
@Component
public class JwtUtil {
    @Value("${app.jwt.secret}")
    private String secret;

    @Value("${app.jwt.expiration}")
    private long expiration;

    public String generateToken(Long userId, String username) {
        return Jwts.builder()
                .subject(username)
                .claim("userId", userId)
                .issuedAt(new Date())
                .expiration(new Date(System.currentTimeMillis() + expiration))
                .signWith(getKey())
                .compact();
    }

    public Claims parseToken(String token) {
        return Jwts.parser()
                .verifyWith(getKey())
                .build()
                .parseSignedClaims(token)
                .getPayload();
    }

    private SecretKey getKey() {
        return Keys.hmacShaKeyFor(secret.getBytes(StandardCharsets.UTF_8));
    }
}
```

#### 4.6.4 JwtInterceptor — JWT认证拦截器

```java
@Component
public class JwtInterceptor implements HandlerInterceptor {

    private final JwtUtil jwtUtil;

    public JwtInterceptor(JwtUtil jwtUtil) {
        this.jwtUtil = jwtUtil;
    }

    @Override
    public boolean preHandle(HttpServletRequest request, HttpServletResponse response, Object handler) {
        String authHeader = request.getHeader("Authorization");
        if (authHeader != null && authHeader.startsWith("Bearer ")) {
            String token = authHeader.substring(7);
            try {
                Claims claims = jwtUtil.parseToken(token);
                Long userId = claims.get("userId", Long.class);
                String username = claims.getSubject();
                UserContext.set(userId, username);
                return true;
            } catch (Exception e) {
                // Token无效
            }
        }
        response.setStatus(401);
        return false;
    }

    @Override
    public void afterCompletion(HttpServletRequest request, HttpServletResponse response,
                                Object handler, Exception ex) {
        UserContext.clear();      // 必须清理ThreadLocal
    }
}
```

### 5.3 Composable 编写规范

#### 5.3.1 useToast — 模块级单例模式

```javascript
// composables/useToast.js
import { ref } from 'vue'

// 模块级状态（所有组件共享）
const toasts = ref([])
let nextId = 0

export function useToast() {
  function show(message, type = 'info', duration = 3000) {
    const id = nextId++
    toasts.value.push({ id, message, type })
    if (duration > 0) {
      setTimeout(() => remove(id), duration)
    }
  }

  function remove(id) {
    toasts.value = toasts.value.filter(t => t.id !== id)
  }

  return {
    toasts,
    toast: {
      success: (msg) => show(msg, 'success'),
      error: (msg) => show(msg, 'error'),
      info: (msg) => show(msg, 'info'),
      warning: (msg) => show(msg, 'warning')
    }
  }
}
```

**规范要点**：
- Toast 状态声明在模块作用域（非函数内），实现全局单例
- 返回 `{ toasts, toast: { success, error, info, warning } }`
- 自动定时移除（默认3秒）

#### 5.3.2 useApi — API请求封装

```javascript
// composables/useApi.js
import { ref } from 'vue'
import { useToast } from './useToast.js'

export function useApi(apiFunc) {
  const loading = ref(false)
  const error = ref(null)
  const data = ref(null)
  const { toast } = useToast()

  async function execute(...args) {
    loading.value = true
    error.value = null
    try {
      const result = await apiFunc(...args)
      data.value = result
      return result
    } catch (e) {
      const msg = e.response?.data?.message || e.message || '请求失败'
      error.value = msg
      toast.error(msg)
      throw e
    } finally {
      loading.value = false
    }
  }

  return { loading, error, data, execute }
}

// 附加：确认删除
export function useConfirmDelete(deleteFn, { toast }) {
  async function handleDelete(id, message = '确定要删除吗？') {
    if (!window.confirm(message)) return
    try {
      await deleteFn(id)
      toast.success('删除成功')
      return true
    } catch (e) {
      toast.error('删除失败: ' + (e.response?.data?.message || e.message))
      return false
    }
  }
  return { handleDelete }
}

// 附加：保存操作
export function useSave(saveFn, { toast }) {
  const saving = ref(false)
  async function handleSave(form, isEdit, loadFn) {
    saving.value = true
    try {
      await saveFn(form)
      toast.success(isEdit ? '修改成功' : '添加成功')
      loadFn?.()
      return true
    } catch (e) {
      toast.error('保存失败: ' + (e.response?.data?.message || e.message))
      return false
    } finally {
      saving.value = false
    }
  }
  return { saving, handleSave }
}
```

#### 5.3.3 useUpload — 七牛云上传

```javascript
// composables/useUpload.js
import { ref } from 'vue'
import * as qiniu from 'qiniu-js'
import { getQiniuToken } from '../data/api.js'

export function useUpload() {
  const uploading = ref(false)
  const progress = ref(0)

  async function uploadImage(file) {
    uploading.value = true
    progress.value = 0
    try {
      const { token, domain } = await getQiniuToken()
      const key = `memory/${Date.now()}_${Math.random().toString(36).slice(2)}.${file.name.split('.').pop()}`
      const observable = qiniu.upload(file, key, token, {}, { uphost: 'https://upload-as0.qiniup.com' })
      return new Promise((resolve, reject) => {
        observable.subscribe({
          next: (result) => { progress.value = Math.round(result.total.percent) },
          error: (err) => { reject(err) },
          complete: (result) => { resolve(`${domain}/${result.key}`) }
        })
      })
    } finally {
      uploading.value = false
    }
  }

  return { uploading, progress, uploadImage }
}
```

#### 5.3.4 useImageCompress — 图片压缩

```javascript
// composables/useImageCompress.js
export function compressImage(file) {
  return new Promise((resolve) => {
    if (file.size < 500 * 1024) {   // 500KB以下不压缩
      resolve(file)
      return
    }
    const img = new Image()
    img.onload = () => {
      const canvas = document.createElement('canvas')
      let { width, height } = img
      const max = 1920
      if (width > max || height > max) {
        if (width > height) { height = height * max / width; width = max }
        else { width = width * max / height; height = max }
      }
      canvas.width = width
      canvas.height = height
      canvas.getContext('2d').drawImage(img, 0, 0, width, height)
      canvas.toBlob(blob => resolve(blob), 'image/jpeg', 0.8)
    }
    img.src = URL.createObjectURL(file)
  })
}
```

**压缩策略**：
- 文件 < 500KB：不压缩，原样返回
- 文件 ≥ 500KB：Canvas压缩，最大 1920px，质量 0.8

```javascript

  response => response,

      window.location.href = '/login'
    }

  }
)

// 接口定义：每个接口函数返回 r.data.data（解包Result结构）
export const login = (username, password) => api.post('/auth/login', { username, password }).then(r => r.data.data)

}
export const createDailyRecord = (data) => api.post('/daily-records', data).then(r => r.data)
```

### 6.4 ViewModel 编写规范

```java
public class DailyViewModel extends ViewModel {

    private final DailyRepository repository;
    private final MutableLiveData<List<DailyRecord>> records = new MutableLiveData<>();
    private final MutableLiveData<Map<String, Object>> stats = new MutableLiveData<>();
    private final MutableLiveData<Boolean> loading = new MutableLiveData<>(false);

    public DailyViewModel() {
        repository = new DailyRepository();
    }

    public LiveData<List<DailyRecord>> getRecords() { return records; }
    public LiveData<Map<String, Object>> getStats() { return stats; }
    public LiveData<Boolean> getLoading() { return loading; }

    public void loadData() {
        loading.postValue(true);
        new Thread(() -> {
            try {
                List<DailyRecord> data = repository.getAllRecords();
                records.postValue(data);
                Map<String, Object> statsData = repository.getStats();
                stats.postValue(statsData);
            } catch (Exception e) {
                e.printStackTrace();
            } finally {
                loading.postValue(false);
            }
        }).start();
    }

    public void createRecord(DailyRecord record) {
        new Thread(() -> {
            try {
                repository.createRecord(record);
                loadData();    // 创建后重新加载
            } catch (Exception e) {
                e.printStackTrace();
            }
        }).start();
    }

    public void deleteRecord(long id) {
        new Thread(() -> {
            try {
                repository.deleteRecord(id);
                loadData();    // 删除后重新加载
            } catch (Exception e) {
                e.printStackTrace();
            }
        }).start();
    }
}
```

**规范要点**：
- 继承 `ViewModel`
- 所有异步操作使用 `new Thread(() -> { ... }).start()`，**不使用协程或RxJava**
- 子线程通过 `liveData.postValue()` 更新UI
- 增删改后调用 `loadData()` 刷新
- Repository在ViewModel构造函数中直接 `new`（不通过依赖注入）

### 6.5 Repository 编写规范

```java
public class DailyRepository {

    private final DailyService dailyService;
    private final AuthRepository authRepository;

    public DailyRepository() {
        dailyService = ApiClient.create(DailyService.class);
        authRepository = new AuthRepository();
    }

    public List<DailyRecord> getAllRecords() throws IOException {
        Integer userId = authRepository.getCurrentUserId();
        Response<ApiResponse<List<DailyRecord>>> response =
            dailyService.getRecords(userId).execute();    // 同步调用
        if (response.isSuccessful() && response.body() != null && response.body().getCode() == 200) {
            return response.body().getData();
        }
        throw new IOException("获取记录失败");
    }

    public void createRecord(DailyRecord record) throws IOException {
        Integer userId = authRepository.getCurrentUserId();
        record.setUserId(userId);
        Response<ApiResponse<Void>> response =
            dailyService.createRecord(record).execute();
        if (!response.isSuccessful() || response.body() == null || response.body().getCode() != 200) {
            throw new IOException("创建记录失败");
        }
    }

    public void deleteRecord(long id) throws IOException {
        Response<ApiResponse<Void>> response =
            dailyService.deleteRecord(id).execute();
        if (!response.isSuccessful() || response.body() == null || response.body().getCode() != 200) {
            throw new IOException("删除记录失败");
        }
    }
}
```

**规范要点**：
- Retrofit **同步调用** `.execute()`（在ViewModel的子线程中执行）
- 统一错误处理：检查 `response.isSuccessful()` + `body().getCode() == 200`
- 抛出 `IOException` 由ViewModel捕获
- Repository在构造函数中创建 `ApiClient.create(XxxService.class)`

## 9. 安全设计规范

### 9.1 JWT认证

| 项目 | 规范 |
|------|------|
| 算法 | HMAC-SHA256 |
| 密钥 | 配置项 `app.jwt.secret` |
| 有效期 | 7天 (604800000ms)，配置项 `app.jwt.expiration` |
| Token内容 | `sub`(用户名), `userId`(用户ID), `iat`, `exp` |
| 传递方式 | `Authorization: Bearer <token>` |
| 拦截范围 | `/api/**`（排除 `/api/auth/login`） |

**认证流程**：
1. 客户端 POST `/api/auth/login` → 获取Token
2. 后续请求 Header 携带 `Authorization: Bearer <token>`
3. `JwtInterceptor` 拦截验证 → 存入 `UserContext` (ThreadLocal)
4. 请求完成 → `afterCompletion()` 清理 ThreadLocal

### 9.2 密码安全

| 项目 | 规范 |
|------|------|
| 加密算法 | BCrypt |
| Bean配置 | `PasswordEncoderConfig` 提供 `BCryptPasswordEncoder` |
| 校验方式 | `encoder.matches(rawPassword, encodedPassword)` |

### 9.3 CORS配置

- 开发环境：允许所有来源 (`allowedOriginPatterns("*")`)
- 生产环境：应限制为前端域名

### 9.4 SQL注入防护

- 所有查询使用参数化（`?` 占位符 + `params.toArray()`）
- 动态条件使用 `List<Object>` 收集参数

### 9.5 前端Token管理

- Web端：`localStorage` 存储，Axios拦截器自动注入
- Android端：`SessionManager` (DataStore + 内存缓存)，`AuthInterceptor` 自动注入
- 401响应：自动清除登录态并跳转登录页

### 10.2 上传流程

```
2. 客户端直传七牛云（不经过后端服务器）
   - 上传路径: memory/{timestamp}_{random}.{ext}
   - Web端: qiniu-js SDK
   - Android端: 七牛云Android SDK
3. 上传完成 → 返回 CDN URL
4. 客户端将URL保存到后端
```
